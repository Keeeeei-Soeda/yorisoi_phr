## 3. AIチャットの「カウンセラー／メンター化」具体設計

### 3-1. 設計原則（5原則）

| # | 原則 | やる | やらない |
|---|---|---|---|
| 1 | **傾聴ファースト** | 受け止め2文・問い1つ／受け止めだけで返すターンも作る | いきなり質問・矢継ぎ早に聞く |
| 2 | **オウム返しと言い換え** | 相手の言葉を具体的に拾う「『朝から手のしびれ』が続いているのは心配ですよね」 | 「そうだったんですね」連発 |
| 3 | **小さく褒める／肯定** | 「気づかれているのが大事ですね」「メモしておこうと思ったの、いいと思います」 | お世辞・大袈裟な褒め |
| 4 | **自己決定を尊重** | 選択肢提示「『書いておく／いったん休む／話を続ける』のどれでも大丈夫ですよ」 | 「○○すべきです」「次はこうしてください」 |
| 5 | **無理に終わらせない** | ユーザーの「保存しよう」「もういいや」が出てから suggestEnd | ターン数で機械的に切る |

### 3-2. システムプロンプト改修案（Before / After）

#### Before（現行）
> あなたは○○の患者さんから「今日の出来事」をやさしく聞き取るチャットボットです。
> - 共感ファースト：いきなり質問せず、まず一言だけ受け止める
> - 1回1質問：1ターンの返信で聞くことは1つだけ
> - 終わらせ方：3回以上話したら「これくらいで大丈夫そうですね」と促す

#### After（改修案・骨子）
```
あなたは○○の患者さんに寄り添う、訓練を受けたピアカウンセラーです。
医療判断は一切しません。代わりに、患者さんが自分の気持ちと体調を
言葉にできるよう、安全な対話を提供します。

## 振る舞いの原則
1. 傾聴ファースト
   - 受け止めだけで返すターンがあってOK
   - 相手の言葉をそのまま使ってオウム返しする（「『朝から重い』が続いているんですね」）
   - 共感の型を毎回変える（「そうだったんですね」連発を絶対に避ける）
2. 小さな肯定を挟む
   - 「気づかれているのが、すでに大事な一歩ですね」
   - 「ここに書こうと思ったの、いいことだと思います」
   - お世辞ではなく、相手の行動を具体的に肯定する
3. 質問は1つ・選択肢を添える
   - 「もう少し教えてもらってもいいですか。たとえば、いつ頃から続いていますか？」
   - 「無理に答えなくて大丈夫です」を時々添える
4. 自己決定を尊重
   - 「○○してください」ではなく「○○もありますし、××もあります」
   - 終わるかどうかは患者さんに決めてもらう
5. 終わらせ方
   - ユーザーが「保存しよう／もういい／ありがとう」を言ったら同意・suggestEnd:true
   - そうでない限り、ユーザーが話したい間は受け止め続ける
   - ターン数で機械的に切らない

## 絶対NG
- 診断・重症度評価・薬の効果判定・「再燃の可能性があります」
- 「○○してください」と命令する
- 「お疲れ様です」「そうだったんですね」など同じ定型句を2回以上使う
- 患者の話を要約して終わらせようとする

## 出力形式（JSONのみ）
{
  "reply": "カウンセラーとしての応答（2〜4文・敬語・絵文字なし）",
  "suggestEnd": <ユーザーが終わりたがっている明確なシグナルがあるかどうか>
}
```

## 4. めんどくさがりが「使いたい・続けたい」を満たす再設計案

### 4-1. 入り口の再設計（最優先）

布谷さん発案の**「何で困ってますか？」フロー**をそのまま借りる。

```
【初回起動 or 久しぶりに開いた時】

  こんばんは。今日はどうされましたか？
  ─────────────────
  □ 今日の体調を残しておきたい
  □ 病院で言われたことをメモしたい
  □ 同じ病気の人と話したい（or 気持ちを聞いてほしい）
  □ 振り返りたい
  □ いつも通り（マイメニューへ）
  □ 何もしないで閉じたい
  ─────────────────
```

**ポイント**:
- 5項目に絞る（**3〜5が選べる上限**・布谷さんの「10は無理」根拠）
- 「**何もしないで閉じたい**」を入れる（ストレス軽減・選んでも罪悪感ゼロ）
- 選択に応じて、出すボタンを変える（**全部見せない**）
- 2回目以降の起動は「いつも通りで開きますか？それとも今日は別件ですか？」のスキップ可

### 4-2. 「最小記録モード」を新設（ベテラン向け）

布谷さんの「**発熱・体重・CRPだけあればいい**」を実装。

```
【最小記録モード ホーム】

   今日の体重    [50.2 kg ▼]
   今日の体温    [36.4 ℃ ▼]
   気になる血便  [なし／少し／多い]
   メモ          [一言だけ]   

   ── 1タップで保存 ──

   グラフ：
     体重・体温 過去30日
     CRP 直近の検査値
```

**ポイント**:
- 1画面で完結
- 設定で「最小モード」をONにすると、ホームがこの画面に切り替わる
- 「いつも通り」を選んだベテランはここに直行

### 4-3. 「何もしないを肯定する」ホーム

寛解期の罪悪感をゼロにする。

```
【ホーム上部・常時表示】

   今日は調子いい日ですか？
   無理して書かなくて大丈夫です。
   調子悪い日に来てくれれば、ここで一緒に整理します。
```

**ポイント**:
- 「連続記録日数」の煽りをやめる、または「お休みOK」表示にする
- 開いてくれたこと自体を肯定する（「今日も来てくれてありがとう」）

### 4-4. 食事記録は「写真ワンタップ・分類しない」

布谷さん発案。社内でも添田さんが「写真撮影ベースで負荷最小化」と合意済み。

```
【食事タブ】

   ［📷 ごはん撮る］← 大ボタン1つ

   過去の食事サムネイル（時系列）
   ─ 2/8 朝   2/8 昼   2/8 夜  …
```

**ポイント**:
- タイトル・カロリー・分類すべて不要
- 体調悪化時に「3日前のラーメン」をサムネ並びで思い出す
- AIに「今週何食べましたか？」と聞かれた時だけ写真をAIに見せる
- **メイン用途は再燃時の振り返り**であって日々の記録ではない

### 4-5. 寄り添う「居場所」を作る（コミュニティの起点）

- 誰にも見せない・流れていく
- AIが愚痴を聞いてくれるボタン（チャット改修と接続）
- 「**話したいだけ／聞いてほしいだけ**」の人の受け皿

**フェーズB（後）**: 患者会連携のアプリ内コミュニティ

## 5. どこを「入り口」にすべきか

### 5-1. 物理的な入り口
LINE Bot内のLIFFアプリなので、**LINEの友達追加が物理的入口**。
- 友達追加直後のWelcome文を再設計
- 「こんばんは。今日はどんなことで開いてくれましたか？」で 4-1 のフローに接続

### 5-2. 心理的な入り口
- 「**困った時に開く**」を前提にする
- 「困ってない時は開かなくていい」と明言する（罪悪感ゼロ）
- これはメインビジュアル・コピー・トーンのすべてに反映

### 5-3. 状態別の入り口
| 状態 | 入り口 |
|---|---|
| 新規発症 | 「何で困っていますか？」5択 → 該当機能 |
| 寛解期ベテラン | 「最小記録モード」直行 |
| 急な体調悪化 | 「ひとりごと／吐き出し」モード |
| 受診前後 | 「病院で言われたこと、メモしておきますか？」 |
| 何もしたくない | 「閉じる」が最初に出ている |

### 5-4. 受信動線（Push）
- 通知は最小限（**通知＝ストレス**）
- 「3日連続で開いていない人」に「無理しなくていいですよ。困った時にいつでも」のソフト通知
- 受診予定があれば「明日◯◯クリニックですね。気になることをメモしておけますよ」
