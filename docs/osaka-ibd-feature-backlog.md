# 大阪IBD向け 機能追加バックログ（叩き台）

- **目的**: 商談中に「これ欲しい」「うちの会員さんはこっちが助かる」を引き出すための素材
- **ステータス凡例**: ⭐実装済 / 🔧軽改修で対応可 / 🆕新機能（要見積もり）

商談時の使い方：「**いまある機能はここまで**。これに加えて、こんな候補を考えています」と一覧で見せる → 反応を聞く → 優先順位を決める。

---

## A. UC/CD 共通の強化候補

### A-1. 🔧 生物学的製剤の投与間隔リマインダー
- いま：`medications` の `sideNotes` にメモを書ける
- やりたい：投与間隔（インフリキシマブ8週・アダリムマブ2週・ステラーラ8週・エンタイビオ8週・ヒュミラ2週・ソリリス2週）を構造化して**次回投与予定日を自動計算・通知**
- なぜ：「うっかり遅れた」が薬剤耐性化のリスクに直結
- 見せ方：ホーム画面に「次のヒュミラまであと3日」のような帯

### A-2. 🆕 便中カルプロテクチン推移グラフの強調
- いま：`lab-tracker` で他検査値と並列表示
- やりたい：UC/CDで**最重要マーカー**として独立カードに昇格、目安ライン（<50正常/250>異常）も視覚化
- なぜ：保険適用も拡大、患者本人の理解の助けに
- 見せ方：「あなたのおなかの炎症の数字」というラベル付きで折れ線

### A-3. 🆕 大腸内視鏡 / MRE 検査の周期管理
- いま：実装なし
- やりたい：「前回 2024-05-15、次回目安 2026-05〜」のように**次回検査の目安**を表示。寛解期は1〜2年毎が標準
- なぜ：寛解期に「忘れる」のが一番のリスク
- 見せ方：あんしんカード隣に「次の内視鏡まで」カード

### A-4. 🆕 患者会イベントカレンダー連携
- いま：実装なし
- やりたい：大阪IBDの**交流会・医療講演会・World IBD Day**を「ふりかえる」カレンダーに重ね表示
- なぜ：会員さまの「次の交流会いつだっけ」を解消、参加率向上にも
- 見せ方：通常の日付の上に「★大阪IBD交流会」のチップ
- **協業ポイント**：大阪IBDからイベント情報をご提供いただく形（HP RSSやGoogleカレンダー連携など）

## B. UC（潰瘍性大腸炎）特化候補

### B-1. 🆕 排便日記の簡略入力（朝・昼・夜のタップ）
- いま：`symptom-log` で1日1回まとめて記録
- やりたい：朝・昼・夜の3タップで**回数＋ブリストルスケール＋血便有無**だけ拾う
- なぜ：UC患者にとって「便の質」が再燃の最早期サイン

### B-2. 🆕 トイレマップ大阪市内強化
- いま：`restroom` 機能あり（汎用）
- やりたい：大阪市内の**多目的トイレ・オストメイト対応・洗浄機付き**を重点表示。会員さまから「この駅のここ」情報を集約
- なぜ：トイレ問題は患者の6割が外出時に意識
- **協業ポイント**：大阪IBD会員さまからの口コミ収集

---

## C. CD（クローン病）特化候補

### C-1. 🆕 栄養療法（成分栄養剤）の継続支援
- いま：`medications` に1薬として登録するのみ
- やりたい：エレンタール / ラコール / エネーボの**1日服用カロリー目標**と達成度トラッキング、フレーバーローテーション提案
- なぜ：CDの寛解維持の柱、しかし続けるのが大変

### C-2. 🆕 体重・BMI 推移グラフ
- いま：症状ログにスケール式（増減のみ）
- やりたい：**実数値の体重**入力で増減トレンドをグラフ化、低下時にアラート
- なぜ：CDの再燃の重要サイン

### C-3. 🆕 肛門病変の経過写真（プライベートメモ）
- いま：visit に `photos` フィールドあり（医師に見せる前提）
- やりたい：**自分専用の経過写真**枠を追加、医師に見せる/見せないを切り替え可能
- なぜ：痔瘻・肛門皮垂の経過を自分で確認したい需要

### C-4. 🆕 狭窄症状アラート（自動受診勧奨）
- いま：CDテンプレに `stenosisSymptoms` トグルあり
- やりたい：3日連続で「狭窄症状あり」がONの場合に**アラート＋受診案内**
- なぜ：腸閉塞の早期検知（CDの代表的緊急合併症）

## D. 大阪IBD固有の協業候補

### D-1. 🆕 「大阪IBDメンバー」モード
- 入会者向けの特別バッジ/挨拶を表示、患者会・支援情報セクションに**大阪IBD独自の連絡先・会員ページへの導線**を追加
- 大阪IBD会報のバックナンバーリンクを設定画面に表示
- **協業ポイント**：会員番号や会員ページとの紐づけ可否を相談

### D-2. 🆕 匿名集計データの患者会への提供
- 多科併診率・受診頻度・寛解期間・困りごとパターン等を**会員のオプトインの上で匿名集計**
- 大阪IBDが行政要望書を出す際の**根拠データ**として活用
- **協業ポイント**：会員さまのプライバシーを守る集計設計を双方で確認

### D-3. 🆕 交流会レポート機能
- 交流会で出た話題・質問を**会員さま自身で投稿→他の会員さまも閲覧**できる仕組み
- 既存の「ふりかえる」と地続きで、自分の体調記録と重ねて見られる
- **協業ポイント**：モデレーション体制が必要（運営とすり合わせ）
