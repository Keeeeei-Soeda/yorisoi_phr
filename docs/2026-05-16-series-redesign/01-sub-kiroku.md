# よりそい きろく — 記録の核

## 1. 一言で

> **急に体調が崩れた時に、過去の自分が助けてくれる。**

「毎日記録する」ことを求めない。「思い出せない時に過去の自分が代わりに覚えていてくれる」存在として記録を位置づける。

## 2. 患者の課題（記録を残せない3つの理由）

| 理由 | 患者の本音 | きろくの答え |
|---|---|---|
| **書くのが面倒** | 「邪魔くさい」（布谷さん） | LINE音声に話すだけ → 系統Bが自動整形してきろくに流入 |
| **何を書けばいいか分からない** | 「下手したら何に困ってるかも本人わかってない」 | 入口AIが「今日はどんな感じですか」と聞き、患者の言葉から記録項目を抽出 |
| **書いても見返さない** | 「日記化リスク」（5/12） | 振り返り画面はキーワード抽出・トレンド表示で「自分の気づき」を返す |

## 3. 3つの記録モード（状態適応）

### 3-1. 急性期/しんどいモード（最小記録）

布谷さん発案「**発熱・体重・CRPだけあればいい**」を発展。

```
今日の体温  [36.4 ℃ ▼]
今日の体重  [50.2 kg ▼]
気になる血便 [なし／少し／多い]    ← IBD系のみ
気になること [一言だけ]
        ↓
   1タップで保存
        ↓
直近30日の体温・体重グラフ
直近のCRP値
```

→ 既存 `/simple/minimal.html` を発展。状態判定で自動的にこのモードに入る。

### 3-2. 安定/寛解期モード（詳細記録）

音声・写真・チャットを使い分けられる。
系統B（LINE音声）と連携して、診察音声から自動的にきろくに整形済み記録が乗る。

```
[🎤 話す] [📷 撮る] [💬 書く]
        ↓
    AI整形（系統B）
        ↓
🧾要点 / 🎯方針 / ✅ToDo / 🚩警告 / 🔎用語やさしい言い換え / ❓次回質問
        ↓
タイムラインに自動追加（visit エンティティ）
```

### 3-3. 食事ワンタップモード

布谷さん発案・添田さん同意「**写真撮るだけ・分類なし・タイトルなし**」。
体調悪化時に「3日前のラーメン」を時系列サムネで思い出すのがメイン用途。

→ 既存 `/simple/meal.html` を発展。

## 4. ターゲット患者像

- **コア**: IBD中重度・パーキンソン・SLE等の進行性疾患患者
- **特徴**: ノートをつけている人、診察で「最近どうですか」に答えるのに困る人
- **避けるユーザー**: 「毎日記録したい」健康管理志向の人（→ 別アプリ推奨）

## 5. 既存資産との対応

| 既存 | きろくでの扱い |
|---|---|
| `public/simple/record.html`（visit入力ウィザード） | 詳細モードのベース |
| `public/simple/minimal.html`（最小記録） | 急性期モードに昇格 |
| `public/simple/meal.html`（食事写真） | 食事モードに |
| `public/restroom.html`（トイレマップ） | 設定下に温存（撤退判断は別途） |
| `server/routes/visits.js` `clinics.js` | データ層として継続利用 |
| 系統B（GitHub/yorisoi/） | `/summary/:sessionId` API化して連携 |

## 6. 主要画面（image生成候補）

1. **ホーム** — 「今日はどんな感じですか？」入口AIが優しく聞く画面
2. **急性期モード** — 大きいボタン3-4個・1タップ保存
3. **詳細モード** — 🎤📷💬の3入力選択
4. **AI整形完了** — 絵文字セクション付き要約が表示される瞬間
5. **タイムライン振り返り** — キーワード抽出・トレンド・気づきカード

## 7. 設計判断（Why）

- **「毎日記録」を求めない理由**: 寛解期患者は「調子いい時こそ忘れる」を罪悪感にする → ストレスを増やすアプリになる
- **系統B連携を主軸にした理由**: 自前で「きろくする」を作るのは重複。系統Bが既に最良の体験を提供している
- **状態適応UI**: 急性期と安定期で必要な機能が真逆。同じUIだとどちらにも中途半端
- **食事写真ワンタップ**: 分類・カロリーは続かない。「再燃時の思い出し」が真の用途

## 8. やらないこと

- 連続記録日数の表示（煽り禁止）
- リマインダー通知の強制（オプトインのみ）
- 「目標体重」「目標歩数」設定（健康管理志向には向かない）
- 食事のカロリー計算・栄養分析（写真ワンタップの思想と矛盾）
