# 入口AI（カウンセラー） — 「今日はどんな感じですか」

## 1. 役割

サブブランド名を最初に見せない優しい入口。
カウンセラーAIが患者の言葉から状態を判定して、適切なサブブランドへそっと誘導する。

5/11布谷さん発案の「**何で困ってますか？**」フローを発展させたもの。
布谷さん指摘「**チャンスの女神は前髪しかない**」を入口AIで掴む。

## 2. 入口の3パターン

### 2-1. 初回起動

```
こんばんは。よりそいへようこそ。

ここは、困った時に来てくれればそれで大丈夫な場所です。
毎日来る必要も、毎日記録する必要もありません。

まずは、今日はどんな感じですか？
よかったら、一言だけ教えてください。
（無理に答えなくても大丈夫です）

[テキスト入力]  または  [何もしない]
```

### 2-2. 2回目以降（最後の利用から1週間以内）

```
おかえりなさい。

  → 前回の続きから  ← よく使う動線
  → 今日はどんな感じですか？  ← カウンセラー入口
  → そのまま閉じる  ← 罪悪感ゼロ
```

### 2-3. 2回目以降（最後の利用から1週間以上）

```
お久しぶりです。来てくれてありがとう。
最近どうされていましたか？よかったら聞かせてください。

  → 話す  → 何もしないで閉じる
```

## 3. 状態判定 → サブブランド誘導

入口AIがユーザーの一言から、以下のように適切なサブブランドに誘導：

| 患者の言葉のシグナル | 推測する状態 | 誘導先サブブランド |
|---|---|---|
| 「しんどい」「お腹が痛い」「熱が」 | 急性期/しんどい | **きろく**（最小記録モード） |
| 「不安」「怖い」「眠れない」「相談したい」 | 心理的苦痛 | **そうだん**（ひとりごとモード or 経験者の声モード） |
| 「明日病院」「主治医に」「医師に伝える」 | 受診前後 | **きろく**（詳細モード）→ **てちょう**（あんしんカード） |
| 「最近どうだろう」「振り返りたい」 | 安定/振り返り | **てちょう**（タイムライン） |
| 「分からない」「○○ってどういう病気」 | 診断理解期 | **そうだん**（診断理解モード） |
| 何も書かない・短い言葉 | 状態不明 | 「もう少し話を聞いてもいいですか？」と問い返し |

## 4. 振る舞いの原則（カウンセラー化）

5/11ドキュメントの5原則を継承：

1. **傾聴ファースト**
2. **オウム返しと言い換え** — 「『朝から重い』が続いているんですね」
3. **小さな肯定** — 「気づかれているのが、すでに大事な一歩ですね」
4. **自己決定を尊重** — 「○○もありますし、××もあります」
5. **無理に終わらせない**

## 5. 「閉じたい」を選んだ時の振る舞い

無理に引き止めない。罪悪感をゼロにする。

```
わかりました。ゆっくり休んでくださいね。
また話したくなったら、いつでも来てください。

  [そのまま閉じる]
```

→ 5/11実装済みの「無理しなくていい」やさしさカードを継承。

## 6. NG（禁忌）

- 状態ラベリング（「しんどい人モードへ」のような可視化されたラベル）
- 「○○すべきです」と命令する
- 同じ定型句の繰り返し
- 病気の重症度判定
- 「3日連続で来てくれてありがとう」のような連続記録煽り

## 7. 実装方針

- 既存 `server/routes/ai.js` の `/api/ai/listen` を発展
- システムプロンプトに「状態判定 → サブブランド誘導」のロジック追加
- フロントは新規 `/index.html` で入口AI画面を実装（既存ホーム置き換え）
- レスポンスJSON: `{ reply: string, suggestedSubBrand: 'kiroku'|'soudan'|'techou'|null, suggestedMode: string|null }`
- サブブランド誘導はあくまで提案（強制しない・ボタンで選べる）

## 8. 設計判断（Why）

- **入口AIにする理由**: 既存「3ボタンドンと並ぶ」は「**何のために開いたか言語化できない人**」を排除していた
- **状態をラベリングしない理由**: 森さんメモリ `feedback_patient_facing_no_labeling.md` —「『しんどい人』NG」「カウンセラー型の迎え方」
- **サブブランド名を最初に見せない理由**: 患者は「自分のしたいこと」で開いていない。「今困っている」で開いている。ブランド選びは負担
- **「何もしない」を最初から提示する理由**: ストレスゼロ。来てくれただけで価値があると伝える
